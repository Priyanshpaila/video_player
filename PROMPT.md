Build a production-ready Next.js App Router fullscreen cinematic video-story microsite with one intro video and five chapter videos. The design must avoid generic AI-style glassmorphism and template-like catalog grids. It should feel closer to an award-style interactive microsite: editorial composition, oversized typography, asymmetric layout, film grain, strong negative space, minimal copy, sharp UI controls, and a memorable immersive scene-selection interaction.

The intro video starts first, muted by default for browser autoplay compatibility. Keep first load almost text-free: only brand overlay, sound toggle, skip control, and a minimal progress line. After the intro finishes or is skipped, show a unique scene-selection page, not a static grid. Use one large active cinematic preview and a scene rail/list with oversized scene numbers. Hover/focus changes the active preview. Click starts the selected video. Keep chapter cards as normal HTML for accessibility and responsiveness.

Once a chapter video is opened, it is considered consumed whether the user watches fully or exits midway. If the user exits midway, show a confirmation dialog explaining the scene will be marked as viewed and removed. After a scene is completed or confirmed exited, return to the selector and show only the remaining scenes. When all five scenes are consumed, show a final completion screen with restart.

Do not save progress in localStorage or sessionStorage. Hard refresh must restart from the intro. Keep real assets in public/videos and public/posters. Provide a mock media mode with cinematic abstract visuals while real videos/thumbnails are unavailable. Use TypeScript, Tailwind CSS, Framer Motion, and an optional isolated Three.js background. The Three.js layer must be easy to disable or remove without affecting the main logic.
